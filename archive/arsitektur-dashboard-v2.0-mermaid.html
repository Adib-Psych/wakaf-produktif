▼ WRITE TO STORAGE
LAYER 2 · FIRESTORE + STORAGE + PROCESSING
💾Cloud Sync + localStorage + Compute Logic
Firestore upstream + 10 storage keys + bridge dictionaries + processing functions
☁️ Upstream Cloud (Firestore Spark plan)
🔥Firestore: wakaf-ngantang/laporan
Cloud collection · receives worker PWA submissions via setDoc(). Real-time read via onSnapshot() listener di monitor-v2. Region: asia-southeast2 (Jakarta). Impact: worker submit → dashboard auto-update <5s.
_deletedTombstones Set
Race-condition guard. Saat Adib delete entry, ID di-add ke Set sebelum Firestore deleteDoc. Listener skip 'added' event untuk tombstoned IDs (prevent re-add via async propagation).
📦IndexedDB Queue (offline fallback)
lapor.html safety net. Online submit push langsung ke Firestore; kalau offline → queue di IndexedDB. maybeAutoSync() drain pada online event + 2s after page load.
🌉 Bridge Function (CRITICAL — Single Translation Point)
🔗normalizeWorkerEntry(data)
SINGLE bridge function untuk translate worker schema ↔ dashboard schema. 12 field aliases:
• jumlah_pohon→pohon · tanggal→tgl · kebun (array → bidang + kebun_list + kebun string) · detail (object → detail_fields + detail=catatan) · rain_24h_mm→rainfall_24h · compliance recompute via computeRainfallAlerts · Glossary tokens via PRUNING_TYPE_BRIDGE + CABANG_*_BRIDGE.
Impact: kalau field baru di lapor.html → WAJIB update normalizeWorkerEntry, kalau tidak silent data loss.
📖Glossary Bridge Dictionaries
3 mapping bahasa awam ↔ token ilmiah:
• PRUNING_TYPE_BRIDGE: "Pangkas berat (total)" ↔ pruning_berat
• CABANG_DIPERTAHANKAN_BRIDGE: "Cabang gemuk berbuah lebat" ↔ cabang_gemuk_berbuah
• CABANG_DIBUANG_BRIDGE: "Cabang ke dalam / cabang balik" ↔ cabang_balik, dll
Token version preserved di detail_fields.*_tokens untuk analisis.
📦 localStorage Keys (10 keys)
STATE.laporan
Daily reports array. Populated dari Firestore listener (worker) + Wakif Form Input. Schema includes detail_fields (preserved dynamic fields).
SOP_STATUS
Status per item SOP_AKTIVITAS_PER_BULAN. Key: ${bulan}-${idx}. Schema v1.2 (pisang removed → one-off task).
UPAH_ITEMS
Pekerja harian + borongan tracker. Auto-create via autoCreateUpahFromEntry (Firestore listener). Status: belum/lunas.
CASHBOOK
Cash flow transactions. Auto-link via batchPay/confirmPaid → ref: 'upah_'+id. Dedupe via Audit Tool guardrail.
MILESTONES
Curated bilingual milestones. Schema v3.0 (implementasi + benchmark_evaluation). Generated via Batch Curation + AI Draft.
HISTORICAL_PHASES
Strategy 1 historikal (replace ANNUAL_COMPLIANCE) — 3 fase pra-formal (Nadzir 1, Nadzir 2, RWI Era). Score retroaktif placeholder.
SOP_AKTIVITAS_PER_BULAN
Master SOP — 12 bulan × ~58 items hardcoded. Oct 2026+ pakai 19 topik A-S (KATEGORI_TOPIK_MAP).
LIVING_LAB_PHASE_MAP
4 fase fenologi (Pre-Panen/Panen/Post-Panen/Awal MH) sync dengan Living Lab. Basis phase-aware Strategy 2.
POHON_BASELINE config
{ pre: 915, post: 750, current: 915, perBidang: [350, 300, 250, 15] }. Update current setelah seleksi pasca-panen 2026.
TIMING_TIER_META + RULES
4-tier classification: STRICT/MODERATE/FLEXIBLE/EVENT-DRIVEN. Multi-Factor Timing scoring basis. La Niña tolerance +14d.

⚙️ Processing Functions (selected)
autoCreateUpahFromEntry()
Trigger di Firestore listener. Input STATE.laporan entry → output UPAH_ITEMS entry (Yit/Kumpul, dedupe by tgl+pelapor+src).
getPhaseComplianceMetrics()
Phase-aware compute (Strategy 2). Per-fase Coverage % + Timing % (multi-factor). Excludes items future.
computeTimingScore()
Multi-factor: tier × days_late × tolerance × ENSO × graduated penalty. Per-item score 0-100.
getKebunIndices()
Multi-kebun aware iteration. Returns array of bidang indices dari kebun_list array OR string 'K1+K3' OR single bidang.
getDetailText()
Defensive helper untuk legacy detail object. Return text version safely (object → catatan fallback).
fixLegacyDetailObjects()
Migration on page load. Detect detail as object → move to detail_fields + set detail = catatan. Persist via localStorage.
reprocessPendingEntries()
Migration 3s after load. Recompute compliance for pending entries + auto-create UPAH for unlinked YIT/KUMPUL entries.
computeAudit() + audit actions
Scan UPAH ↔ Cashbook for duplicates/orphans/syncs. Actions: auditUnmarkPaid() + auditResetAllPaid() + auditDeleteCashbook().
