Methodology Score Retroaktif

Score(fase) = Σ (item × weight × match_score) / Σ (item × weight)
match_score: 100% (sesuai SOP) · 50% (substituted/approximate) · 0% (tidak ada)
weight: wajib 3× · rekomendasi 2× · rutin 1×
📋 Filosofi Strategy 1: Score historikal tidak dimaksudkan sebagai blaming era sebelumnya. Tujuan utama adalah:
  1. Akuntabilitas — angka konkret untuk laporan ke umat/donor
  2. Pelajaran — pattern apa yang konsisten miss (mis. brand substitution tanpa BRIN consultation di era RWI)
  3. Baseline calibration — tahu starting point sebelum SOP formal sehingga improvement Mei 2026+ punya konteks

Lesson Learned Insights — Era Pra-Formal

3 pola yang muncul dari Fase 2-3-4 (untuk dihindari di era Self-Managed):

  1. Sparse documentation — output panen tercatat (KG-level) tapi proses agronomi tidak tracked. Lesson: minimal viable daily log adalah baseline akuntabilitas.
  2. Brand substitution tanpa BRIN consultation — premium brands (NPK Yaramila, Karate Plus Boron, Fish Extract) substitute SOP BRIN spec (Urea + SP-36 + KCl + Kieserit + Dolomit). Lesson: substitusi merk → lock di MoU baru kalau memang necessary, jangan asumsi setara.
  3. Tidak ada SOP-level tracking — output (panen 281 KG) ≠ compliance tinggi karena tidak ada systematic per-aktivitas check. Lesson: monitor process-level, bukan hanya output.

Scoring Formula (Per Fase)

Phase Score = (sudah ÷ items_due_so_far_excluding_NA) × 100%
items_due_so_far = items yang seharusnya sudah dikerjakan di bulan ≤ bulan berjalan, dalam fase tsb

Kenapa Bukan Linear YTD?

Methodology lama (pre-v2.1) hitung: compliance% = items_sudah ÷ items_total_kalender_YTD. Masalah:

⚠️ Anti-pattern linear YTD: Items pre-panen sedikit (~15) tapi pre-panen pendek (Mei-Jul 3 bulan); items pemupukan MH banyak (~20) tapi MH-nya November. Hasilnya: di Mei-Jul score akan stuck rendah meski semua pre-panen items sudah selesai, karena denominator = total 60 items setahun. Wakif tidak akan dapat sinyal perbaikan sampai akhir tahun.

Phase-aware fixes ini: per-fase punya score sendiri yang mature di akhir window-nya. Pre-panen 100% achievable bulan Juli (saat fase selesai), bukan Maret tahun depan.

⚡ Quality Dimension Split (Detail)

Sub-Component A — Tier Alignment (15% of total)

Quantifies adherence to SOP procedural standards:

100% Aligned — sesuai SOP penuh
75% Approximate — ~10% deviation acceptable
50% Substituted — metode alternatif (off-SOP)
30% Partial — separuh eksekusi
0% Deviation — tidak comply

Sub-Component B — Convergence Status (10% of total) 🆕

NEW sejak v2.0: quantifies confidence level based on 4-way validation Trifecta SOP v2.1 (NGT + Puslitbang + Lampung BRMP + International Tier 1-5):

100% KONVERGEN PENUH / FREKUENSI — full validation
100% UNIK UNIVERSAL — Indonesia-unique, universally validated
75% KONVERGEN BASELINE — lower confidence baseline
60% DIVERGEN (Mg/Dosis/Paradigma) — perlu adaptasi lokal
50% KONVERGEN BAHAYA — flagged risk (e.g., pisang)
30% CRITICAL — perlu validation BRIN (Excelsa, Pestisida)
Aturan Kombinasi: Quality final = (Tier Alignment × 0.60) + (Convergence Status × 0.40). Atau lebih simple: Tier kontribusi 15 poin max + Convergence 10 poin max → total 25 poin max dalam Quality 25%.

⏱️ Multi-Factor Timing Engine

Timing 20% sekarang multi-factor, bukan simple binary "ada/belum". Function computeTimingScore(item, status, bulanItem, currentMonth, currentYear, isLaNina) di monitor-v2.html compute per-item score 0-100 dengan 5 faktor:

Factor 1: Tier Criticality

Determined by getTimingTier(item) — explicit tag timing_tier atau auto-classify via pattern matching. Default fallback: flexible untuk items new/un-tagged.

Factor 2: Days Late Calculation

dueDate = mid-month (day 15) of item's bulan
daysLate = max(0, today − dueDate) in days

Factor 3: Tolerance Window (ENSO-adjusted)

🟡 MODERATE±28 hari±42 hari
🟢 FLEXIBLEN/AN/A
⚡ EVENT-DRIVEN±14 hari (dari trigger)±14 hari (climate-neutral, trigger-based)

Factor 5: ENSO State Integration

NOAA ONI (Oceanic Niño Index) fetched dynamic dari psl.noaa.gov/data/correlation/oni.data:

  • ONI ≥ +0.5 (3-month avg) = El Niño → tolerance +14d untuk STRICT & MODERATE
  • ONI ≤ −0.5 (3-month avg) = La Niña → tolerance +14d untuk STRICT & MODERATE
  • −0.5 < ONI < +0.5 = Neutral → tolerance standard

Cache 24 jam di localStorage. Manual override available di UI banner kalau Wakif punya local intel berbeda (mis. BMKG Karangploso forecast lebih akurat untuk Ngantang microclimate).

🎚️ 4-Tier Classification System

Setiap item SOP_AKTIVITAS_PER_BULAN di-classify ke 1 dari 4 tier:

🔴 STRICT
Window Biologis Sempit
Miss = yield/quality impact signifikan. Tolerance ±14d (±28d La Niña). Late > 2× = 0% credit.
🟡 MODERATE
Preferred Window
Shift 2-6 minggu tanpa major impact. Tolerance ±28d (±42d La Niña). Late > 2× = 25% credit.
🟢 FLEXIBLE
Rutin / Ongoing
Tidak ada window strict. Eksekusi kapan saja = 100% credit. No late penalty.
⚡ EVENT-DRIVEN
Trigger-Based
Trigger: cherry merah 80%, OPT > threshold, dll. Tolerance ±14d dari trigger. Late = 50% credit.

📊 Dual KPI Display (Coverage + Timing per Fase)

Phase card di SOP Compliance tab sekarang display 2 angka berdampingan, bukan blended single yang ambigu:

Coverage = ratio items sudah dieksekusi / items due dalam fase (excluding N/A).
Timing = average computeTimingScore() result untuk items due dalam fase.

Contoh fase Pre-Panen (Mei-Jun 2026, fase berjalan):

  • Coverage 93%: 14 dari 15 items due sudah dieksekusi
  • Timing 87%: rata-rata score timing 87% — 12 items on-time (100%), 2 items late dalam tolerance (75%), 1 item out-of-tolerance (45%)

Wakif lihat keduanya → tahu fase ini eksekusi bagus tapi sedikit late di subset items. Bukan single number ambigu "compliance 89%" yang tidak actionable.

❌ TIDAK Termasuk Documentation Scoring (optional meta):

Optional metadata berikut nice-to-have untuk audit trail dan tetap di-encourage, tapi tidak masuk hitungan scoring:
  • Foto umum aktivitas (selfie pekerja, foto progress)
  • Voice note kronologi (kecuali yg explicit SOP-required)
  • Catatan kualitatif free-text (kecuali yg essential context untuk SOP output)
  • Timestamp activities (auto-captured by system)
  • GPS coordinates (auto-captured)
Filosofi: Pak Kumpul yang eksekusi pemupukan 100% sesuai BRIN dosis (Topik C) tanpa foto = score 100% Quality + 100% Documentation (kalau pemupukan tidak ada dokumentasi SOP-mandated). Foto = nice but not penalty.

🔍 Scoring Logic Documentation Dim

100% All SOP-mandated docs delivered + readable
70% Docs delivered tapi incomplete (mis. partial lab result)
50% Docs ada tapi tidak readable / hilang sebagian
0% SOP-mandated docs missing entirely
N/A Aktivitas tidak punya SOP-mandated doc → 100% (no penalty)
Aturan operasional: Saat audit aktivitas, evaluator pertama cek "apakah SOP minta dokumentasi specific?". Kalau YA → score per kriteria di atas. Kalau TIDAK → otomatis 100% (no penalty for missing optional meta). Ini menjaga methodology tetap honest tentang execution quality vs paper trail completeness.
