// pseudo-code rumus upah (di monitor-v2.html)
function autoCreateUpahFromEntry(entry) {
  const tarif = WORKER_TARIF[entry.pelapor]?.[entry.durasi];
  if (!tarif) return; // pelapor not in registry (mis. ANTO post-OUT)
  if (entry.kategori_borongan) tarif = TARIF_BORONGAN[entry.kategori] * entry.output_qty;
  UPAH_ITEMS.push({
    id: uid(), tgl: entry.tgl, pelapor: entry.pelapor, kategori: entry.kategori,
    durasi: entry.durasi, tarif, src: 'auto_form', status: 'unpaid',
    laporan_id: entry.id, cashbook_id: null
  });
  saveUpah();
}

// dispatch when "Bayar" clicked
function markPaid(upahId) {
  const u = UPAH_ITEMS.find(x => x.id === upahId);
  u.status = 'paid'; u.paid_at = new Date().toISOString();
  const cashId = autoCreateCashbookFromUpah(u); // src='autolink'
  u.cashbook_id = cashId;
  saveUpah(); saveCashbook(); recomputeRABAbsorption(); refreshKPI();
}

🔄 23.1 · Pattern #10 — Firestore Sync Global Variable (Locked 20 Mei 2026)

Casus belli: Pemupukan recovery 20 Mei butuh inject 2 scheduled reminder task ke Tab Tim & Distribusi Tugas fase Panen. Pakai pattern lama (mutate localStorage saja) → log "✓ Pushed to Firestore" muncul, tapi data tidak update di Firestore (false positive). Round-trip verify gagal.

🐛 Root Cause

Sync functions seperti syncDistribusiPicToFirestore(), syncSOPMasterToFirestore() baca dari global in-memory variable (mis. DISTRIBUSI_DATA), BUKAN dari localStorage. Kalau hanya update LS lalu call sync, function push global (un-modified) → Firestore tetap stale. Console log tetap success (karena Firestore write API memang succeed, hanya data-nya yang sama).

✅ 4-Step Recipe (Wajib Berurutan)

// 1. Mutate global in-memory variable FIRST
DISTRIBUSI_DATA[idx].aktivitas.push(newEntry);

// 2. Persist to localStorage (defensive)
saveDistribusiToLS();

// 3. Push to Firestore (function reads from global)
await syncDistribusiPicToFirestore();

// 4. Re-render UI
renderDistribusi();

🔍 Verification Round-Trip

Setelah sync, call load<Collection>FromFirestore(). Kalau data hilang → push tidak commit (silent fail). Pattern naming reusable: sync<CamelCase>ToFirestore() umumnya reads dari <ALL_CAPS>_DATA global.
