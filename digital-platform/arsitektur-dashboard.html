▼ WRITE TO STORAGE
LAYER 2 · STORAGE + PROCESSING
💾localStorage + Compute Logic
7 storage keys + processing functions — semua client-side, no server backend
📦 Storage Keys (localStorage)
STATE.laporan
Daily reports array. Auto-populate dari Form Input. v3 schema dgn id + src tags.
SOP_STATUS
Status per item (Belum/Sudah/Tidak Sesuai/N/A). Key: ${bulan}-${idx}. Seeded Jan-Apr dari Living Lab.
UPAH_ITEMS
Pekerja harian + borongan tracker. Auto-link dari Form Input untuk Yit/Kumpul.
CASHBOOK
Cash flow transactions. Auto-link saat batchPay upah lunas → CASHBOOK entry.
MILESTONES
Curated public milestones bilingual. Schema v3.0 dgn implementasi + benchmark_evaluation.
PETTY_CASH_ANTO
Petty cash Mas Anto sebagai koordinator lapangan. Track pengeluaran ad-hoc.
ANNUAL_COMPLIANCE
Historical scores 2024 (12%) · 2025 (16%) · 2026 (dynamic blend).
SOP_AKTIVITAS_PER_BULAN
Master SOP data — 12 bulan × 58 aktivitas hardcoded. Source: SOP Robusta Edisi 2 + BRIN.
⚙️ Processing Functions
getComplianceMetrics()
Compute aggregate dari SOP_STATUS (sudah/tidak/belum/na counts + overdue/upcoming items).
computeAnnualCompliance(year)
2026 YTD blend = (sudah + 0.5×tidak_sesuai) / total · partial credit formula.
renderComplianceAnalysis()
Dynamic analysis "kenapa rendah" — list belum wajib + tidak_sesuai dgn SOP guidance.
5-Dimension Scoring
Coverage 30% + Quality 25% + Timing 20% (La Nina ±1bln) + Cost 15% + Doc 10%.
